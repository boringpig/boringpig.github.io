Git語法認識

語法 描述
git config –list 查看本機端git的config的設定
git init 初始化資料夾為git的repository
git add . 新增所有檔案到git
git add 新增指定的檔案到git
git commit -m 對於新增的檔案發出commit
git status 查看目前staged data的狀態
git log 查看commit之後的所有歷史紀錄
git diff 查看檔案被修改的內容
git clone 從遠端的repo複製到本機端的資料夾

Branch認識


master branch : 只接受 hotfix 和 release 兩個 branch 的 merge，為最後發佈的版本

  • hotfix branch : 用來處理已釋出版本需要修改錯誤，最後在 merge 回來並標上發佈的版本

develop branch : 所有的 feature 都從這邊分支出去，最後在 merge 回來

  • release branch : 凍結 develop 的 function 進行測試和修改，作為 master 釋出的版本 tag
  • feature branch : 主要為develop開發新增的功能。
Pull Request : 用來 fork 別人的 repo 並clone到本機端，建立一個 branch 新增或修改程式再發佈 Request 給他人使用者，確認後merge並deploy
