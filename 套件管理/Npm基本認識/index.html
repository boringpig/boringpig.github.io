NPM基本認識

2016-07-27
套件管理

NPM不僅是nodejs的套件管理，也是前端框架套件管理的工具，相關的也有像是bower、composer等等。要寫好一個完整的網頁，有些功能不知道怎麼寫、或是資料夾結構要怎麼編排等等，一開始可以先套用別人寫好的套件來使用，等熟悉流程、重點要素，就可以自己開發，因為畢竟使用別人寫好的套件想要客製化，也要熟悉他人套件的使用流程、資料結構等等。

Version

Node 6.5.0
NPM 3.10.3

安裝方式

安裝NPM很簡單，現在只要去Node.js官方網站，下載安裝Node.js也會一起安裝NPM

基本語法

語法 描述
npm init 初始化資料夾並產生一個新的package.json
npm install npm@latest -g 將npm安裝到最新版
npm info 查看安裝套件的訊息內容
npm search 從npm倉庫搜尋套件
npm install 目前路徑下安裝
npm install -g 全域安裝
npm install –force 強制重新安裝一次
npm update 版本更新
有時候會因為套件衝突，只想要安裝特定的版本，可以下這條指令   `npm install @<版本號>`

版號認識

[主版本號].[次版本號].[修訂版本號]，例如：Bootstrap 3.1.1

主版本號 : 軟體有重大更新，例：功能或介面大幅更動。
次版本號 : 軟體發佈新功能，但不會大幅影響整個軟體。
修訂版本號 : 當軟體有 bug，發布 bug 的修正版

當發佈測試版本主版本號會是0，例0.1.0

安裝位置

我們開發有分為兩種階段，一個是develop開發階段，另一個是production產品階段，也就是說產品階段不需要debug套件之類等等。

--save : 套件名將被添加到dependencies(產品階段使用)
--save-dev : 套件名將被添加到devDependencies(開發階段使用)。

套件腳本

有些套件管理附有 script 可以自定義腳本指令，也就是說當你輸入npm start 執行在瀏覽器上，會去跑你這個專案所需要的套件，像是：資料庫連線之類的。
