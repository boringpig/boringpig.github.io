Npm基本認識

2016-07-27

NPM

安裝連結

  • NodeJS下載

    常用語法

  • npm init 根目錄會初始化生成一個新的package.json文件
  • npm install npm@latest -g 將npm安裝到最新版
  • npm info <package> 查看安裝套件的訊息內容
  • npm search <package> 從npm倉庫搜尋套件
  • npm install安裝套件
    • npm install <package> 本地安裝
    • sudo npm install -g <package> 全域安裝
    • npm install git://github.com/package/path.git
    • npm install <package> --force 重新安裝一次
  • npm update版本更新

    安裝不同版本

  • npm install <package>@<版本號>
  • [主版本號].[次版本號].[修訂版本號]
    • 主版本號: 軟體有重大更新，例：功能或介面大幅更動。
    • 次版本號: 軟體發佈新功能，但不會大幅影響整個軟體。
    • 修訂版本號: 當軟體有 bug，發布 bug 的修正版

  • 當發佈測試版本主版本號會是0，例0.1.0

    安裝位置

  • --save：套件名將被添加到dependencies(上線階段使用)。
  • --save-dev: 套件名將被添加到devDependencies(開發階段使用)。

    npm run

  • 用來執行package json文件裡的scripts字段的腳本命令

  • npm run lint 會去執行 lint 指令

  • npm run 會去執行scripts裡面的所有指令
  • npm start、npm test 屬於特殊指令不需加 run，分別代表開始編譯、測試

Git

安裝

  • git下載
  • SourceTree下載

    語法認識

語法 描述
git init 在local初始化repository
git add . 新增所有檔案到git
git add Readme.md 新增指定的檔案到git
git commit -m 對於新增的檔案發出commit
git commit -am 對於舊有的檔案修改後發出的add、commit
git status 查看目前git狀態
git log 查看目前git的所有歷史紀錄，包括commit、add過的檔案等等
git diff 查看檔案被修改的內容
git clone 從remote端複製到local端的git

Git Flow

  • master branch : 只接受 develop 和 Release 的 merge，為最後發佈的版本。
    • hotfix branch : 用來處理已釋出版本需要修改錯誤，最後在merge回來並標上發佈的版本。
  • develop branch : 所有的feature都從這邊分支出去，最後在merge回來。
    • release branch : 凍結develop的function進行測試和修改，作為master釋出的版本tag。
    • feature branch : 主要為develop開發新增的功能。
  • Pull Request : 建立一個branch描述新功能在發佈Request，確認後才merge，最後再deploy程式碼。
